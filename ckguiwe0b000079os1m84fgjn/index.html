MySQL8的新特性

发表于 2020-10-29 分类于 MySQL 阅读次数：

MySQL的版本一下子跳跃这么大，用手指想就有很多新的东西了。
下面我们就来详细了解一下。

1. 秒级加列

  1. 分别在5.7与8.0中有一张 test 表，数据为3000w行
    # mysql8.0.22
mysql> alter table test add str varchar(200) not null default 'mysql8.0 新加字段';
Query OK, 0 rows affected (0.13 sec)
Records: 0  Duplicates: 0  Warnings: 0

# mysql5.7.30
mysql>  alter table test add str varchar(200) not null default 'mysql5.7 新加字段';
Query OK, 0 rows affected (6 min 8.36 sec)
Records: 0  Duplicates: 0  Warnings: 0
    可以看到 8.0 只需要 0.13 秒，而 5.7 需要 6分多钟！
    以前表加列操作需要重建表（消耗大量的IO资源和时间），8.0加列没有这个步骤。

2. 性能提升


# mysql5.7.30
mysql> select count(*) from test;
+----------+
| count(*) |
+----------+
| 29991137 |
+----------+
1 row in set (3 min 12.24 sec)

# mysql8.0.22
mysql> select count(*) from test;
+----------+
| count(*) |
+----------+
| 30000000 |
+----------+
1 row in set (45.70 sec)

可以看到三千万的数据量，8.0版本只需要45.70秒，而5.7版本需要3分12.24秒
分别在只更新和只读场景下，8.0相比5.7在高并发时性能提升近1倍；求表总数据量8.0响应时间也将近提升5倍。

3. 文档数据库

  • MySQL Document Store，NoSQL + SQL = MySQL
  • 多文档事务，ACID特性
  • 支持更新JSON中部分字段

4. SQL增强

  • 公用表表达式（CTE）
    关于CTE的详细介绍看这里 https://developer.aliyun.com/article/719994
  • 不可见索引（Invisible Indexes）
    将索引置为不可见时，SQL执行时查询优化器会忽略该索引；这样有助于我们了解该索引对查询语句的影响有多大。
    -- 将索引置为不可见
alter table test alter index  index_name invisible;
-- 置为可见
......................................... visible;
  • 降序索引（Descending Indexes）
    在5.7的时候只支持升序索引，就算显式指定为降序，创建的还是升序。
    具体例子 降序索引详解
  • 函数索引（Functional Indexes）
    在之前的版本中我们知道，凡是涉及到计算以及函数的都无法使用索引。 那么现在这已经是过去式了
    select * from test where year(date_) = 2020;
    而且函数索引可以和普通索引组合为联合索引，就像普通索引与普通索引组合那样，相同的方法

5. 默认字符集utf8mb4

mysql root@localhost:(none)> show variables like '%character%';
+--------------------------+--------------------------------+
| Variable_name            | Value                          |
+--------------------------+--------------------------------+
| character_set_client     | utf8                           |
| character_set_connection | utf8                           |
| character_set_database   | utf8mb4                        |
| character_set_filesystem | binary                         |
| character_set_results    | utf8                           |
| character_set_server     | utf8mb4                        |
| character_set_system     | utf8                           |
| character_sets_dir       | /usr/share/mysql-8.0/charsets/ |
+--------------------------+--------------------------------+
8 rows in set
Time: 0.028s
  • 可以更好的存储emojis
  • 可变长度编码字符的性能提升

6. 一致性查询改进

  • SKIP LOCKED
    需要加锁的记录若被其它线程占有锁，则跳过，而不是等待

    select * from test where age=12 for update skip locked;
Empty set (0.00 sec)

  • NOWAIT
    需要加锁的记录如果有锁，则会报错

    select * from test where age=12 for update nowait;
ERROR 3572 (HY000): Statement aborted because lock(s) could not be acquired immediately and NOWAIT is set.

    这两个特性某种意义上替代了redis的作用。

7. 资源组

  • 线程赋给不同的资源组
  • 资源组和不同的内存、IO、CPU(现仅支持)进行关联

8. 新的数据字典

  • 基于innodb的库表元数据信息
  • 增强了MySQL crash-safe能力
  • 原子DDL（ Atomic DDLs ）

    以前版本MySQL数据字典存放在多个地方，一机器多实例时存在大量文件描述符性能消耗。

8.0版本都存放在事务性InnoDB表，MySQL异常挂掉后也不会再出现表损坏情况；DDL操作失败也不会再留下占空间的临时文件

9. MGR 增强

  • 金融级别99.999%官方高可用方案
  • MGR是业务多活(应用多活+数据库多活)的终极方案
  • 多个MySQL组成一个group，数据写group
  • 线上10套多主8.0 MGR集群

    MGR的增强大大提升了在网络异常(机房级故障)下的健壮、稳定性

13. 全局变量持久化

以前我们在客户端执行MySQL配置，都是暂时性的，MySQL重启后就会失效。想永久设置只能修改配置文件。8.0版本直接通过SQL命令，就可以直接永久配置。

  • set persist
    会在设置当前全局变量的前提下，将其保存到 mysqld-auto.cnf 文件中，这个命令需要有这个权限：system_variables_admin，才能执行
  • set persist_only
    只会将其保存到 mysqld-auto.cnf 文件中，而不会设置当前全局变量，这对只读全局变量有用，这个命令需要有这个权限：persist_ro_variables_admin，才能执行
