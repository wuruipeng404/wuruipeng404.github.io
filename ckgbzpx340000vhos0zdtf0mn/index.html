Golang程序实现自动更新

发表于 2020-10-16 分类于 Golang 阅读次数：

    我们都知道一个正常的golang程序部署上线的流程大致如下，编译为二进制文件，上传服务器运行。当然这些流程可以通过自动化发布平台去自动完成，当然如果你使用docker的话，则是去构建镜像，当然这一流程也可以继承到CI/CD平台中去自动化完成。用K8S就更简单了，扯远了。。。

假如我们就想实现一个不登录服务器，去通过暴露一个接口让程序自动更新他自己呢？

  1. go run + 监控文件变动
    之前有看到一个库是用来监测go源代码变动的，如果有变动则会使变动实时生效，不过这个库是什么我忘了。这种方式比较奇葩。因为这样的话就意味着你的go程序 是没有编译，使用go run的形式跑在服务器的。这种我还不如用python对吧。当然了python程序也可以编译后在运行呢。

  2. jpillora/overseer
    这是一个开源的组件，用来实现上述需求。同类型的项目还有https://godoc.org/github.com/cloudflare/tableflip,这里我们暂时不研究他。

    • 原理：
      通过fork一个子进程去执行main函数中的程序逻辑，所有的请求会通过当前的主进程（也就是 监听进程），通过队列通道下放到子进程（也就是我们自己真正的进程）中取执行 。

    • 废话少说：撸代码。

      package main

import (
  "fmt"
  "github.com/gin-gonic/gin"
  "github.com/jpillora/overseer"
  "github.com/jpillora/overseer/fetcher"
  "io/ioutil"
  "net/http"
  "time"
)

var (
  BuildID string = "0"
)

func main() {
  overseer.Run(overseer.Config{
    Program: program,
    Address: ":8080",
    Fetcher: &fetcher.HTTP{
      URL:      "http://localhost:4000/bin/app",
      Interval: 3 * time.Second,
    },
    Debug: true,
  })
}

func program(state overseer.State) {
  r := gin.Default()
  r.GET("/", func(c *gin.Context) {
    c.JSON(200, gin.H{"正在运行版本": BuildID})
  })

  // 通过接口上传更新后的版本，触发overseer去自动更新
  r.Any("/bin/app", func(c *gin.Context) {
    if b, err := ioutil.ReadFile("path/main"); err != nil {
      c.JSON(200, gin.H{"msg": err.Error()})
    } else {
      c.Writer.WriteHeader(http.StatusOK)
      c.Header("Content-Disposition", "attachment; filename=main")
      c.Header("Content-Type", "application/octet-stream")
      c.Header("Content-Length", fmt.Sprintf("%d", len(b)))

      if c.Request.Method == "HEAD" {
        c.Header("ETag", "v002")
        c.Status(200)
      } else {
        _, _ = c.Writer.Write(b)
      }
    }
  })

  _ = r.RunListener(state.Listener)
}
      • 注意: 像此类接口，实际开发过程中一定要注意鉴权，以及校验上传文件的哈希值，确保万无一失。哎，见过太多没有安全意识的小盆友了。
      • 缺陷: 监听者自身的配置，还是需要手动更新，无法实现自动更新。
      • 虽然花了一点时间研究，不过在K8S，docker这么好用的今天，这个我也没在实际项目使用，纯当记录吧
