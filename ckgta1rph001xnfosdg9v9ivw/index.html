理解设计模式之装饰器模式

发表于 2020-10-26 分类于 设计模式 阅读次数：

熟悉python的朋友对装饰器肯定不陌生，他的定义就是在不改变原有定义的情况下，动态的添加功能。
这里的装饰器模式其实也是一个意思

  1. 首先来看一个最简单的,在go中函数是一等公民，所以是可以当成参数传递的，那么这就简单了。

    func pprint() {
	fmt.Println("1234")
}

func pprintDecorator(a func()) {
	fmt.Println("decorator")
	a()
	fmt.Println("after")
}

func main() {
	pprintDecorator(pprint)
}

  2. 再来看一个标准的

    package main

import (
	"fmt"
	"log"
)

// 定义一个API, API中只有一个方法就是计算获取结果
type API interface {
	Calc() int
}

type Object struct{}

func (*Object) Calc() int {
	return 0
}

// 乘 装饰器
type MulDecorator struct {
	API
	num int
}

func WarpMulDecorator(c API, num int) API {
	return &MulDecorator{
		API: c,
		num: num,
	}
}

func (d *MulDecorator) Calc() int {
	return d.API.Calc() * d.num
}

// 加 装饰器
type AddDecorator struct {
	API
	num int
}

func WarpAddDecorator(c API, num int) API {
	return &AddDecorator{
		API: c,
		num: num,
	}
}

func (d *AddDecorator) Calc() int {
	return d.API.Calc() + d.num
}

func main() {
	obj := &Object{}
	d := WarpAddDecorator(obj, 100)
	log.Println(d.Calc())
}
