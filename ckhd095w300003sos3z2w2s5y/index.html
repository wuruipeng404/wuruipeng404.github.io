python调用rust与go生成的库文件

发表于 2020-11-11 更新于 2020-11-12 分类于 rust 阅读次数：

Python,Rust,Go 我最喜欢的三门语言。

今天来搞一下 分别用rust&&go生成.so文件供python调用，顺便简单测试一下他们之间的性能差距

版本

  • Python: 3.8.6
  • Go: 1.15.3
  • Rust: 1.49.0-nightly (25f6938da 2020-11-09)

1. Rust

  1. 编写python能用的库文件，我们需要用到pyo3这个项目（其实不用也可以，但是会比较吃藕，pyo3可以直接包装成python的数据类型,以及python的module。而且pyo3的功能不止如此）
    1. 安装
      rust版本分为 stable, beta 和 nightly 版本，pyo3目前需要使用nightly版本，所以我们先安装 nightly
      rustup install nightly
# 如果已经安装过则升级
rustup update nightly

# 切换
rustup default nightly
    2. 修改.cargo配置文件
      vim ~/.cargo/config
      填入如下内容
      [target.x86_64-apple-darwin]
rustflags = [
  "-C", "link-arg=-undefined",
  "-C", "link-arg=dynamic_lookup",
]
  2. 创建项目
    cargo new r2py
  • 创建好后目录下会有一个Cargo.toml文件，内容如下
    [package]
name = "r2py"
version = "0.1.0"
authors = ["rumple"]
edition = "2018"

[dependencies]

# 以下为手动添加依赖与配置部分
[lib]
name = "r2py"
crate-type = ["cdylib"]

[dependencies.pyo3]
version = "*"
features = ["extension-module"]
  1. 编写库文件
    在src下创建lib.rs文件,我们来创建一个计算函数
    use pyo3::prelude::*;
use pyo3::wrap_pyfunction;


#[pyfunction]
fn add(num: u64) -> PyResult<u64> {
    let mut sum = 0;

    for i in 0..=num {
        sum += i;
    }
    Ok(sum)
}

#[pymodule]
fn r2p(_py: Python, m: &PyModule) -> PyResult<()> {
    m.add_wrapped(wrap_pyfunction!(add))?;
    Ok(())
}
  2. 编译：cargo build --release
    编译后会在src同级目录生成 target/release/libr2py.dylib,你也可以直接指定cargo配置文件中的crate-type去直接生成so文件，不过这里是一样的。
  3. 把文件拷贝到我们的python测试目录待用。
    cp target/release/libr2py.dylib  ${yourdirectory}/r2py.so

2. Go

  • 编写 libadd.go

    package main

import "C"

//export add
func add(num uint64) uint64 {

  var (
    i, sum uint64   
  )

  for i < num {
    i++
    sum += i
 }
  return sum
}

func main()  {
  
}

  • 编译

    go build -buildmode=c-shared -o libadd.so libadd.go

3. 编写Python测试脚本

import time
import r2py # pyo3的好处。可以直接当做module导入

from ctypes import cdll, c_ulonglong

go_lib = cdll.LoadLibrary("./src/libadd.so")
# 处理数据类型 不然会溢出
go_add = go_lib.add
go_add.restype = c_ulonglong


NUM_CONST = 1000000000


def time_decorator(func):
    def wrap():
        start = time.process_time()
        a = func()
        end = time.process_time()
        print(f"cost:{end - start} seconds")
        return a

    return wrap


@time_decorator
def py_test():
    s = 0
    for i in range(NUM_CONST + 1):
        s += i
    return s


@time_decorator
def rust_test():
    return r2py.add(NUM_CONST)


@time_decorator
def go_test():
    return go_add(NUM_CONST)


if __name__ == '__main__':
    print("=" * 50)
    print(f"python 结果:{py_test()}")

    print("=" * 50)
    print(f"rust 结果:{rust_test()}")

    print("=" * 50)
    print(f"go 结果:{go_test()}")

4.结果

结果显而易见啦，不过这里让我惊奇的是 go 竟然比 rust 还要快一点，golang 牛逼。
当然啦这里只是一个很简单的计算场景，孰强孰弱还需要更多不同场景的探索,而且他们所适用的方向可能也不太一样。
但rust毕竟是号称新时代的底层语言，我觉得潜力还是很大的。
