k8s之Heketi(四)

发表于 2021-01-06 分类于 k8s 阅读次数：

前言

在上一篇中，我们搭建了GlusterFS分布式存储集群。其实搭建好之后就可以创建PV以及PVC进行使用了。但是K8S更进一步的提供一个高级抽象StorageClass, 他主要的作用是对用户设置的PVC申请屏 蔽后端存储的细节，一方面减少了用户对于存储资源细节的关注，另一方面减轻了管理员手工管理PV的工作，由系统自动完成PV的创建和绑定，实现了动态的资源供应。

Heketi

heketi是一个提供RESTful API管理gfs卷的框架，能够在kubernetes、openshift、openstack等云平台上实现动态的存储资源供应，支持gfs多集群管理，便于管理员对gfs进行操作，在kubernetes集群中，pod将存储的请求发送至heketi，然后heketi控制gfs集群创建对应的存储卷。
heketi动态在集群内选择bricks构建指定的volumes，以确保副本会分散到集群不同的故障域内。
heketi还支持任意数量的glusterfs集群，以保证接入的云服务器不局限于单个glusterfs集群。

显而易见 如果我们要使用gfs驱动的StroageClass，就需要通过Heketi了。

安装部署

  • 我们继续在K8S当中进行部署 heketi 服务,创建heketi.yaml

    apiVersion: v1   # 分配一个账户以供使用
kind: ServiceAccount
metadata:
  name: heketi-service-account

---
kind: Deployment
apiVersion: apps/v1
metadata:
  name: deploy-heketi
  labels:
    glusterfs: heketi-deployment
    deploy-heketi: heketi-deployment
spec:
  selector:
    matchLabels:
      name: deploy-heketi
  replicas: 1
  template:
    metadata:
      name: deploy-heketi
      labels:
        name: deploy-heketi
        glusterfs: heketi-pod
    spec:
      nodeSelector:
        heketi: "yes"
      serviceAccountName: heketi-service-account
      containers:
        - name: deploy-heketi
          image: heketi/heketi:latest
          ports:
            - containerPort: 8080
          env:
            - name: HEKETI_CLI_SERVER
              value: http://localhost:8080
          volumeMounts:
            - mountPath: "/var/lib/heketi"
              name: db
            - mountPath: "/etc/heketi"  # 如果要自定义配置文件可以进行挂载
              name: config
          readinessProbe:
            timeoutSeconds: 3
            initialDelaySeconds: 3
            httpGet:
              port: 8080
              path: "/hello"
          livenessProbe:
            timeoutSeconds: 3
            initialDelaySeconds: 30
            httpGet:
              port: 8080
              path: "/hello"
      volumes:
        - name: db
          hostPath:
            path: "/heketi-data"
        - name: config
          hostPath:
            path: "/heketi-config"
---
kind: Service
apiVersion: v1
metadata:
  name: deploy-heketi
  labels:
    glusterfs: heketi-service
    deploy-heketi: support
spec:
  selector:
    name: deploy-heketi
  ports:
    - port: 8080
      name: deploy-heketi
      targetPort: 8080

  • 执行 kubectl apply -f heketi.yaml

  • 创建 GFS 集群

    • 首先编写topology.json文件拷贝并进入到 heketi pod 当中
      {
  "clusters": [
    {
      "nodes": [
        {
          "node": {
            "hostnames": {
              "manage": [
                "k8s-master"
              ],
              "storage": [
                "192.168.0.1"
              ]
            },
            "zone": 1
          },
          "devices": [
            "/gluster-data"
          ]
        },
        {
          "node": {
            "hostnames": {
              "manage": [
                "k8s-node-1"
              ],
              "storage": [
                "192.168.0.2"
              ]
            },
            "zone": 1
          },
          "devices": [
            "/gluster-data"
          ]
        },
        {
          "node": {
            "hostnames": {
              "manage": [
                "k8s-node-2"
              ],
              "storage": [
                "192.168.0.3"
              ]
            },
            "zone": 1
          },
          "devices": [
            "/gluster-data"
          ]
        }
      ]
    }
  ]
}
    • 执行 heketi-cli --user admin --secret 'My Secret' topology load --json=topology.json
    • 查看集群状态 heketi-cli --user admin --secret 'My Secret' topology info

  • 创建 StorageClass

    apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: gluster-heketi
provisioner: kubernetes.io/glusterfs # 必须
parameters:
  resturl: "http://ClusterIP:Port"
  restuser: "admin"

  • PVC申请

    kind: PersistentVolumeClaim
apiVersion: v1
metadata:
  name: pvc-storage-class
spec:
  storageClassName: gluster-heketi
  accessModes:
    - ReadWriteMany
  resources:
    requests:
      storage: 1Gi
